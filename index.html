05

Claude Code とは？

ブラウザのチャットとの違いは、この2つだけ押さえればOK。

フォルダにアクセスできる

ファイルを直接読んで、直接書ける。「このプロジェクトの続きを作って」が成立する。

ターミナルから動かす

自分のPC上で動く。npm install も git commit も、Claude Code がやってくれる。

Twitterっぽいシンプルなアプリを作って。

いきなり作らずに、まずプランモードで計画を見せて。
プロンプト 02 / プランへの追加要望 
ログイン機能はなし、データはブラウザ内のみで保存、
1ファイルのHTMLで動くようにして。
コツ 1

プランモードを使う

AIは速い。だから 最初に止める。
いきなり書かせず、まず「何を作るつもりか」を言わせる。

やり方

Shift + Tab を2回押すとプランモードに入る。緑色のバッジが出る。

作法

返ってきた計画を読む。要らない機能を削る。納得したら Approve、違えば書き直しさせる。

プランを見た瞬間に「あ、これ違うな」と気づけることが多い。
書かれてから直すより、書かれる前に直す方が圧倒的に速い。

コツ 2

フォルダ構造で一貫性を保つ

モックが膨らむと、AIはプロジェクトを読み切れなくなる。
フォルダを整えておくと、AIが迷子にならない。

今日は「そういうものがある」だけ覚えて帰ればOK。詳しくは第2回でやります。

最小の型

my-app/
  README.md    ← 何を作っているか
  src/         ← コード
  data/        ← データ

README には何を書く？

「このアプリは何で、次にAIに何を頼みたいか」を1画面ぶん。次にClaude Codeを開いた自分が読んで思い出せる粒度。

プロンプト 03 / フォルダ整理 
このプロジェクトのフォルダ構造とREADMEを整えて。
次回また続きを作るので、引き継げる状態にしてほしい。
コツ 3

AIっぽさを消す

AIに作らせると、出力に独特の「におい」が出る。3つだけ意識すると、ぐっと自然になる。

  1. 絵文字を使わない

    見出しにもボタンにも本文にも入れない。それだけでほぼ「AIっぽさ」は消える。

  2. 機能を盛りすぎない

    AIは頼まなくても通知・設定・プロフィール…と足してくる。1画面1目的を明示的に指示する。

  3. コピーから「革新的・最適化・あなただけの」を消す

    ダミーデータの "Sample Project Alpha" や "John Doe" も、そのままだと一発でAI製とバレる。

角丸や色などのUI論には今日は踏み込まない（脱線するので）。1つだけ言うなら「全部を rounded-2xl にしない」くらい。

§ 宿題

小さいモックアップを1個

次回までに、何でもいいので自分のPCでモックアップを1つ作ってきてください。
完成させなくていい。途中で詰まった状態で持ってきてくれるのが一番。

07

宿題の要件

  • Claude Code で作る（プランモードから始める）
  • フォルダを切って、README を1枚置く
  • 完成させなくてOK。詰まった箇所を覚えておく
  • テーマは自由。思いつかなければ次のセクションから選んでください
08

迷ったら: 記録系アプリ3案

「毎日1つ足していく」だけのアプリは、AIとの相性が良くて、詰まりにくい。

体重・体調ログ

日付と数値だけ。体重、睡眠時間、気分を数字で残す。グラフは要らない。

読書記録

タイトル、読了日、5段階評価、一言メモ。並べて眺めるだけで満足度が高い。

飲んだコーヒー記録

豆、淹れ方、店、点数、メモ。趣味の記録は自分だけの資産になる。

この3つ以外でもOK。
「自分が本当に欲しかったやつ」を作るのが一番手が動きます。

10

次回予告 / 第2回 もくもく会

2週間後、みんなの宿題を持ち寄って、詰まりをその場で解消します。

  • README や引き継ぎ資料を毎回読ませて、設計思想を崩さない工夫
  • フォルダ構造を綺麗に保つ（肥大化するとAIが読み切れなくなる問題）
  • 要求 → 要件定義 → 出力 のサイクルをうまく回す指示の出し方
  • （余裕があれば）Vercel / GitHub Pages でのホスティング、Supabase の紹介
