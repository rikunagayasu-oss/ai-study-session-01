05

Claude Code とは？

ブラウザのチャットとは別物の道具です。違いは3つ。

チャットじゃなく“エージェント”

チャットは話を返すだけ。Claude Code はファイルを開く・書き換える・コマンドを実行する。「作って」でほんとに作る。

プロジェクトを丸ごと理解する

添付したファイルだけじゃなく、フォルダごと読む。既存コードの“続き”が書ける。

入り口は選べる（中身は同じ）

Terminal / VS Code / JetBrains / デスクトップアプリ / ブラウザ / iOSどこから入っても、エンジンは共通。CLAUDE.md や MCP の設定はどの入り口でも効く。

公式: “agentic coding tool that reads your codebase, edits files, runs commands, and integrates with your development tools.” docs.claude.com/en/docs/claude-code/overview

コツ 1

プランモードを使う

AIは速い。だから 最初に止める。
いきなり書かせず、まず「何を作るつもりか」を言わせる。

やり方

Shift + Tab を2回押すとプランモードに入る。緑色のバッジが出る。

作法

返ってきた計画を読む。要らない機能を削る。納得したら Approve、違えば書き直しさせる。

不明点は聞いてくれる

「Twitterっぽいアプリ」だけ渡すと、AI側から「投稿は誰に見える？」「データはどこに保存？」と質問が返ってくる。プロンプトに「不明点は質問して」と書いてもいいけど、プランモードだと自然にそれをやってくれるのが大きい。

プランを見た瞬間に「あ、これ違うな」と気づけることが多い。
書かれてから直すより、書かれる前に直す方が圧倒的に速い。
聞いてくれるので、こちらから全部指示しなくてよくなる。

コツ 2

思想とルールを書いておく、フォルダを整える

CLAUDE.md はAIエージェントに“このPJで大切にしていること”を毎回思い出させる仕組み。
フォルダ構造は、それを「読ませる単位」として整えておく。

Claude Code は起動時にプロジェクト直下の CLAUDE.md を毎回読んでくれる。README.md に書いても効くけど、公式パターンは CLAUDE.md。今日は簡単のため「READMEと同じ役割のもの」と思ってOK。

CLAUDE.mdはAIに書かせる

自分で書かなくていい。「このPJのCLAUDE.mdを整えて」と頼めばよい。

“思想”と“ルール”を両方書く

思想の例: 「水族館記録アプリでは競争を煽らない。比較・ランキング機能は提案しない」→ 下流の機能提案でランキングを引っ込めるようになる。

ルールの例: 「返答は日本語」「レコードは削除せず archive フラグを立てる」「ライブラリ追加は先に相談」。

思想は判断のガード、ルールは手癖のガード。両方あると効きが違う。

フォルダは“読ませる単位”

肥大化するとAIが読み切れなくなる。最小構造を早めに切って、迷子にさせない。

my-app/
  CLAUDE.md    ← 思想とルール
  src/         ← コード
  data/        ← データ
コンテキスト = AI エージェントの“短期記憶”
会話が進むほど覚えていることが増える。増えすぎると効率が落ち、少なすぎると忘れる。
新しいセッションを開始した時に CLAUDE.md を読ませるのは、その短期記憶を必要な分だけ育てる動作。フォルダと CLAUDE.md を整えておくと、育てるのも絞るのもどちらもやりやすい。

このプロジェクトのフォルダ構造とCLAUDE.mdを整えて。

コツ 3

AIっぽさを消す

AIに作らせると、出力に独特の「におい」が出る。3つだけ意識すると、ぐっと自然になる。

  1. 絵文字を使わない

    見出しにもボタンにも本文にも入れない。それだけでほぼ「AIっぽさ」は消える。

  2. 機能を盛りすぎない

    AIは頼まなくても通知・設定・プロフィール…と足してくる。1画面1目的を明示的に指示する。

  3. メインカラーを最初に1色決めておく

    色を任せるとカラフルになりがち。「主役は #ff2d16、あとはグレー」と最初に渡すだけで、AIっぽいカラフルさが消える。

AIは“見た目”を見ていない
HTMLコードとして書いているだけなので、改行や余白の詰まり方は下手。
スクショや画面操作を挟むと品質は上がるが、その分コンテキスト・トークンは膨らむ（トレードオフ）。

実例: この後に出てくるプロンプト04 / 宿題スターターの3行、読むと3行目に条件が詰め込まれた感じ、気づきますか？ これがまさに“AIの改行”です（あえて直さず残しています）。

08

迷ったら: 記録系アプリ3案

「毎日1つ足していく」だけのアプリは、AIとの相性が良くて、詰まりにくい。

体重・体調ログ

日付と数値だけ。体重、睡眠時間、気分を数字で残す。グラフは要らない。

読書記録

タイトル、読了日、5段階評価、一言メモ。並べて眺めるだけで満足度が高い。

飲んだコーヒー記録

豆、淹れ方、店、点数、メモ。趣味の記録は自分だけの資産になる。

この3つ以外でもOK。
「自分が本当に欲しかったやつ」を作るのが一番手が動きます。
